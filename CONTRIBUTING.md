# 🤝 Contributing to BigBrain Academy

먼저 이 프로젝트에 기여해주셔서 감사합니다! 🎉

## 기여 방법

### 🐛 버그 리포트

버그를 발견하셨나요? 다음 정보를 포함하여 Issue를 작성해주세요:

- **버그 설명**: 어떤 문제가 발생했나요?
- **재현 방법**: 단계별로 설명해주세요
- **예상 동작**: 어떻게 작동해야 하나요?
- **실제 동작**: 실제로는 어떻게 작동하나요?
- **환경**: 브라우저, OS 등
- **스크린샷**: 가능하다면 추가해주세요

### 💡 기능 제안

새로운 기능을 제안하고 싶으신가요?

1. 먼저 [Issues](../../issues)를 검색하여 중복 제안이 없는지 확인
2. Issue를 작성하여 기능 제안
   - 어떤 문제를 해결하나요?
   - 어떻게 구현될 수 있나요?
   - 사용 사례는 무엇인가요?

### 🔧 Pull Request

코드를 직접 기여하고 싶으신가요?

1. **Fork** 이 저장소
2. **Clone** 본인의 fork
   ```bash
   git clone https://github.com/YOUR_USERNAME/BigBrainAcademy-LC.git
   ```
3. **브랜치 생성**
   ```bash
   git checkout -b feature/amazing-feature
   ```
4. **변경사항 작성**
   - 코드 스타일 유지
   - 주석 추가
   - 테스트 (가능하다면)
5. **커밋**
   ```bash
   git commit -m "Add: 놀라운 기능 추가"
   ```
6. **Push**
   ```bash
   git push origin feature/amazing-feature
   ```
7. **Pull Request 생성**
   - 변경 내용 설명
   - 관련 Issue 번호 언급

## 📝 코딩 컨벤션

### JavaScript
- 들여쓰기: 2 spaces
- 세미콜론: 사용
- 따옴표: 작은따옴표 또는 큰따옴표 일관성 유지
- 함수명: camelCase
- 주석: 복잡한 로직에는 설명 추가

### HTML/CSS
- 들여쓰기: 4 spaces
- 클래스명: kebab-case
- 의미있는 class명 사용

### Commit Messages
- `Add:` - 새 기능 추가
- `Fix:` - 버그 수정
- `Update:` - 기존 기능 업데이트
- `Refactor:` - 코드 리팩토링
- `Docs:` - 문서 수정
- `Style:` - 코드 스타일 변경

## 🧪 테스트

변경사항을 제출하기 전에:
1. 사용자 페이지 테스트
   - 회원가입 / 로그인
   - 인증 추가
   - 통계 확인
2. 관리자 페이지 테스트
   - 대시보드 로드
   - 사용자 관리
   - 리포트 생성
3. 브라우저 호환성
   - Chrome
   - Firefox
   - Safari

## 📋 체크리스트

PR을 제출하기 전에 확인하세요:

- [ ] 코드가 정상 작동함
- [ ] 기존 기능이 깨지지 않음
- [ ] 주석이 적절히 추가됨
- [ ] README 업데이트 (필요한 경우)
- [ ] 커밋 메시지가 명확함

## 💬 질문이 있으신가요?

- Issue를 통해 질문하세요
- 기존 Issue와 PR을 먼저 확인해보세요
